# Название проекта
Разработка Telegram-бота для защиты документов с использованием цифровых водяных знаков

Программа реализует Telegram-бота, который позволяет защищать документы (PDF, DOCX, изображения) с помощью цифровых водяных знаков. Бот поддерживает нанесение видимых и скрытых меток, проверку подлинности документов и контроль целостности файлов через SHA-256 хэширование.

# Как запустить

## 1. Скачайте проект 

```bash
git clone https://github.com/mychinasa23/watermark-telegram-bot.git
cd watermark-telegram-bot
```

## 2. Создайте виртуальное окружение

```bash
# Windows
python -m venv .venv
.venv\Scripts\activate

# Linux / macOS
python3 -m venv .venv
source .venv/bin/activate
```

## 3. Установите зависимости

```bash
pip install -r requirements.txt
```

## 4. Создайте файл .env с токеном (через терминал или вручную)

### Вариант через терминал 
```bash
echo BOT_TOKEN=ваш_токен_от_BotFather > .env
echo DB_PATH=bot_database.db >> .env
echo TEMP_DIR=temp_files >> .env
```

### Или создайте файл .env в блокноте и вставьте:
```text
# BOT_TOKEN=ваш_токен_от_BotFather
# DB_PATH=bot_database.db
# TEMP_DIR=temp_files
```

- Как получить токен: Напишите @BotFather в Telegram → /newbot → укажите имя бота → скопируйте токен

## 5. Запустите бота

```bash
python bot.py
```

## 6. Проверьте работу

- Откройте Telegram
- Найдите вашего бота по имени
- Нажмите "старт"

# Структура проекта 

```
watermark-telegram-bot/
│
├── bot.py                 # Точка входа. Запускает бота
├── config.py              # Настройки. Загружает токен из .env
├── utils.py               # Утилиты. Хэширование, очистка файлов
├── requirements.txt       # Список библиотек для установки
├── .env                   # Токен бота (создаётся локально)
│
├── handlers/              # Обработчики команд
│   ├── start.py           # Приветствие и меню
│   ├── watermark.py       # Нанесение водяного знака
│   ├── verify.py          # Проверка водяного знака
│   ├── integrity.py       # Проверка целостности
│   └── history.py         # История документов
│
├── services/              # Работа с форматами файлов
│   ├── wm_image.py        # Обработка JPG/PNG
│   ├── wm_pdf.py          # Обработка PDF
│   └── wm_docx.py         # Обработка DOCX
│
├── database/              # База данных
│   └── db.py              # SQLite (пользователи, документы)
│
└── temp_files/            # Временные файлы (создаётся автоматически)
```
